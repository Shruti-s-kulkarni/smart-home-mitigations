# Threat: Physical Access

| **Primary threat vector** | **Second order threat vector ** | **Mitigating security controls** | **Impact** | **Zero Trust Policy and Mitigations to be applied a smart home owner** |
| :---                      |    :----:                           |          :---:                   | :---:      |  :---:                                                                 |
|                           |                                     |                                  |
| The Ethernet cable gets damaged accidently or deliberately      | None  | None       | The smart devices stop working till the cable is connected again | Least Privilege Policy  Restricting access to the cable to unauthorised people. Secure the cable in a covering that is tamper resistent |
|                           |
| Downgrading of security controls due to owner's lack of knowledge | None | Vendors publishing application security controls and their purposes for owners such that any deviations can be detected by the owners | Privacy breach | Comprehensive visibility with a) security markings on devices, if available and b) education and awareness about security controls |
|                           |
| Malicious actors locating vulnerable people in smart home environments  | Malicious profiling via smart meters or smart home devices like camera that are available on public network without any security controls or weak security controls | None | Fraud, Disabling alarms | Comprehensive visibility with education and awareness about asking for identification before letting people in |
